In Pass 1, your group will read your scenario closely and generate your own observations, concerns, and questions.

## Part B: Identify the Learning Mode

For your chosen scenario, decide whether the model appears to be fixed, adaptive, or hybrid.

| Learning Mode | Working Definition                                                                  |
| ------------- | ----------------------------------------------------------------------------------- |
| Fixed         | Calibrated before deployment and not substantially changed unless manually updated. |
| Adaptive      | Updated over time based on what the device encounters in use.                       |
| Hybrid        | Allowed to adapt within preset limits or only after review.                         |

Answer:

| Question                                                      | Response |
| ------------------------------------------------------------- | -------- |
| Which learning mode seems most plausible for this scenario?   |          |
| What evidence from the scenario supports your interpretation? |          |
| What would change if the model were fixed?                    |          |
| What would change if the model were adaptive?                 |          |
| What would change if the model were hybrid?                   |          |

## Part C: What Becomes Possible?

Think about the value proposition.

| Question                                                                                  | Response |
| ----------------------------------------------------------------------------------------- | -------- |
| Why might edge processing matter in this scenario?                                        |          |
| Why might low power matter in this scenario?                                              |          |
| Why might low latency matter in this scenario?                                            |          |
| What local patterns could the chip learn?                                                 |          |
| What kinds of drift or change might matter over time?                                     |          |
| Would the same benefit be possible with a centralized cloud-based system? Why or why not? |          |

## Part D: What Could Go Wrong?

Think about risk, failure, misuse, and unintended consequences.

| Question                                                                  | Response |
| ------------------------------------------------------------------------- | -------- |
| What happens if the chip learns the wrong pattern?                        |          |
| What happens if it adapts to a harmful, unsafe, or unfair baseline?       |          |
| What happens if the model changes in ways no one can reconstruct?         |          |
| What happens if the chip becomes too sensitive?                           |          |
| What happens if the chip is not sensitive enough?                         |          |
| What happens if people over-trust the chip because it seems personalized? |          |

|                               |                        |                            |

Consider:

<!-- .compact -->

- Who is directly monitored?
- Who benefits if the chip works well?
- Who is harmed if the chip makes a mistake?
- Who receives or acts on the chip's alerts?
- Who owns or controls the device?
- Who maintains, updates, audits, or disposes of it?
- Who is affected indirectly?

## Part F: What Happens After an Anomaly Is Flagged?

The same chip has very different implications depending on what happens after it detects something unusual.

| Question                                                                                                                                 | Response |
| ---------------------------------------------------------------------------------------------------------------------------------------- | -------- |
| Does the chip only record a signal?                                                                                                      |          |
| Does it alert a human?                                                                                                                   |          |
| Does it automatically change a device or system?                                                                                         |          |
| Does it trigger discipline, denial of service, medical intervention, military action, emergency response, or another high-stakes action? |          |
| How much time does a human have to review the decision?                                                                                  |          |
| What would a false positive cost?                                                                                                        |          |
| What would a false negative cost?                                                                                                        |          |
