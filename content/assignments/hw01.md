Practicing anticipatory governance through a substantive group case-study analysis.

## Overview

In this project, you will work in a group of three to analyze one possible future use of a neuromorphic edge chip designed for anomaly detection and possible continuous learning.

This assignment is a **warm-up for the final group project**. Later in the course, your group will choose your own AI-mediated system to analyze. For now, each group will practice the same method using a different BRAID-related scenario.

Your task is not to decide whether the technology is simply "good" or "bad." Your task is to ask:

> What becomes possible? What could go wrong? Who is affected? And what should be required before this technology is deployed?

This project is designed to help you move between:

- technical mechanisms
- social and ethical implications
- governance questions
- concrete recommendations

## Group Structure

- You will work in groups of **3 students**.
- Each group will choose or receive **one scenario**.
- Each group will complete the analysis in **three passes**.
- Each group will submit **one artifact per pass**.
- Each group will submit a **final written brief**.
- Each group will give a **short presentation**.

## Core Learning Goals

By the end of this assignment, you should be able to:

- Explain how technical features of an AI system create governance questions.
- Analyze anomaly detection as a social and technical process.
- Identify how continuous learning can create both benefits and risks.
- Distinguish between a fixed model and a model that adapts after deployment.
- Identify stakeholders who may benefit from, be harmed by, or be responsible for an AI system.
- Use an anticipatory governance framework to organize ethical, legal, and societal concerns.
- Make a specific governance recommendation before a technology becomes widely deployed.
- Present a clear, evidence-based case to classmates.

## Key Terms

| Term | Working Definition |
| ---- | ------------------ |
| Anomaly detection | Identifying examples, signals, or events that differ from what a system has learned to expect. |
| Edge computing | Processing data locally on or near a device rather than sending everything to a central server. |
| Neuromorphic computing | Computing inspired by features of biological nervous systems, often involving spikes, timing, local memory, or brain-inspired hardware. |
| Continuous learning | A system's ability to update what it has learned after deployment as it encounters new data. |
| Fixed model | A model whose behavior is fixed after training unless it is deliberately retrained or updated. |
| Adaptive model | A model that updates after deployment based on new data from its environment. |
| Hybrid model | A model that can adapt within preset limits or only after review. |
| Drift | Gradual change in a person, machine, environment, or data pattern that makes an earlier model less reliable over time. |
| Anticipatory governance | A way of asking social, ethical, legal, and policy questions before a technology becomes locked into everyday use. |

## How the Assignment Works

Your group will complete the project in **three passes**.

The point of the three-pass structure is to help you move from first reactions to organized analysis to a concrete governance recommendation.

| Pass | Purpose | Required Artifact |
| ---- | ------- | ----------------- |
| Pass 1 | Generate observations, concerns, and questions before using the framework. | Scenario worksheet |
| Pass 2 | Organize and deepen your concerns using the analytic framework. | Framework sorting table and top-three category selection |
| Pass 3 | Turn your analysis into a governance recommendation and written brief. | Final brief and presentation notes |

You should complete the passes in order. Do not skip directly to the final brief.

## Deliverables

Your group will submit the following:

### 1. Pass 1 Artifact: Scenario Worksheet

Submit your completed Pass 1 worksheet. This should show your group's first observations, questions, and stakeholder notes before using the analytic framework.

### 2. Pass 2 Artifact: Framework Sorting Table

Submit your completed Pass 2 framework sorting table. This should show how your group sorted your original concerns into the six framework categories and selected your three priority governance concerns.

### 3. Pass 3 Artifact: Final Brief

Submit a polished group brief that includes:

1. A brief summary of the scenario.
2. The value proposition of the chip in this scenario.
3. The main technical features that create governance concerns.
4. The stakeholders and affected parties.
5. Three priority governance concerns.
6. One governance recommendation.
7. One anticipatory question that should be asked before deployment.

### 4. Group Presentation

Each group will give a short presentation to the class.

Your presentation should:

- explain the scenario
- identify the value proposition
- describe the most important technical mechanism
- name the three priority governance concerns
- present your recommendation
- end with one unresolved anticipatory question

## Individual Reflection

In addition to the group submission, each student should answer:

1. What is one technical feature of the chip that changed how you thought about the governance problem?
2. What is one concern your group did not notice until you used the analytic framework?
3. What is one question you would want BRAID researchers, policymakers, or affected communities to discuss before deployment?
