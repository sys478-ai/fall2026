## Topic / Focus

This topic clarifies that machine learning is not one single method. Systems can learn from labeled examples, detect patterns without labels, update through interaction, or rely on simpler rule-based logic that is sometimes described as intelligent even when it is not learned at all.

The point is not to turn the course into a large survey. It is to help students understand that different paradigms make different assumptions about what data exists, what counts as success, and what kinds of outputs a system is designed to produce.

By the end of this topic, you should be able to compare several kinds of machine learning and explain why the word learning can hide important technical differences.

## Guiding Questions

- What is the difference between supervised and unsupervised learning?
- When does a system rely on rules rather than learned patterns?
- What kinds of tasks fit different learning paradigms?
- Why do these distinctions matter for how people imagine AI?
- How do technical differences change later governance questions?

## In This Class

In this meeting, we will:

- compare supervised, unsupervised, and rule-based approaches
- discuss what kinds of data each approach requires
- identify what each paradigm can and cannot do well
- prepare for later questions about decisions, thresholds, and prediction

## BRAID / ELSI Connection

BRAID sits inside a broader AI landscape that includes many different technical paradigms. This topic helps keep students from overgeneralizing from one kind of AI system to another, especially when comparing neuromorphic systems with more familiar machine learning models.
